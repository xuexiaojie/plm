# 14 项目资料与 AI 联合分析规格

## 1. 目标

V1.0 支持按项目录入工程资料，并将同一设备的计算结果与资料上下文提交给大模型 API 做联合分析。

## 2. 资料类型

| code | 中文名 | 典型内容 |
|---|---|---|
| `site_feedback` | 现场反馈 | 安装、调试、运行、问题反馈 |
| `drawing_review` | 审图单 | 图纸会审、审查意见、设计校核意见 |
| `technical_attachment` | 技术附件 | 技术协议、参数表、说明附件、计算附件 |
| `drawing_catalog` | 图纸目录 | 图纸清单、版本目录、专业图纸索引 |

## 3. 录入方式

1. 单条录入：通过 `POST /api/projects/{project_id}/artifacts` 创建一条资料。
2. 批量录入：通过 `POST /api/projects/{project_id}/artifacts/batch` 一次创建多条资料。
3. 批量录入必须支持四类资料混合提交。
4. `project_item_id` 可为空；为空时表示资料属于项目级资料。
5. 资料内容 V1.0 使用文本字段保存，后续文件中心可迁移到附件存储。

## 4. AI 联合分析

AI 联合分析请求以 `equipment_name` 为分析主线。例如同样的“装出钢机”，可联合以下上下文：

1. 计算执行记录和计算结果。
2. 现场反馈。
3. 审图单。
4. 技术附件。
5. 图纸目录。

系统必须将上述上下文组织为结构化 JSON，再生成 Prompt 调用大模型 API。

## 5. 三流分析维度

AI 联合分析必须引入物质流、能量流、信息流三个维度：

1. 物质流：分析坯料、设备、输送节拍、装出炉路径、现场物流衔接与计算对象的一致性。
2. 能量流：分析温度制度、热量供给、保温时间、表里温差、热效率和能量利用风险。
3. 信息流：分析现场反馈、审图单、技术附件、图纸目录、计算输入和版本信息之间的一致性。

输出结果必须包含三流分析、风险点、矛盾点和建议。

## 6. 大模型 API 配置

环境变量：

```bash
export AI_API_URL="https://example.com/v1/chat/completions"
export AI_API_KEY="your-api-key"
export AI_MODEL="your-model-name"
```

未配置 `AI_API_URL` 或 `AI_API_KEY` 时，系统返回本地演示分析，并将 Provider 标记为 `mock`。

## 7. 验收要求

1. 批量录入四类资料后，资料列表可查询到 4 条记录。
2. AI 联合分析可引用至少 1 条计算结果和 4 类资料。
3. AI 分析返回结果必须包含 Provider、状态、三流分析和分析内容。
4. 测试环境未配置大模型 API 时必须稳定返回 Mock 分析。
