# 00 版本范围规格

## 1. 目标

本文件定义工业炉计算平台 V1.0 的实现边界、验收范围和后续规划边界。代码生成必须以本文件为第一优先级约束。

## 2. V1.0 必做范围

| 功能 | 实现要求 | 验收要求 |
|---|---|---|
| 项目管理 | 单点录入、批量录入、候选名单、台账查询 | 可按项目名称、项目经理、企业和智能关键词查询 |
| 名目管理 | 创建、列表、绑定炉型、软删除 | 可按步进炉、辊底炉、环形炉分类管理计算名目 |
| 计算树 | 根节点、目录节点、计算节点、排序 | 名目下可展示一棵树 |
| 计算模板 | 三类炉型模板 | 模板包含输入输出 Schema |
| Python 执行器 | 同步执行 | 保存执行记录、结果、日志 |
| 执行快照 | 输入、输出、模板版本、执行器版本 | 历史结果可复现 |
| 审批 | 提交、通过、退回 | 状态机合法流转 |
| 报告 | 草稿、待审、正式报告 | 审批通过后生成正式编号 |
| 横向对比 | 按 `step_type` 聚合结果 | 同类结果可对比 |
| 项目资料管理 | 现场反馈、审图单、技术附件、图纸目录 | 支持单条和批量录入 |
| AI 分析 | 调用大模型 API，未配置时演示回退 | 同一设备可联合计算和资料分析 |
| 权限 | 六类角色 | API 和按钮按角色控制 |
| 种子数据 | 用户、角色、字典、样例项目 | 初始化后可直接演示 |

## 3. V1.0 不做范围

1. C# 执行器正式运行。
2. 外部可执行程序执行器。
3. 异步任务队列。
4. 独立前端工程。
5. 文件中心完整能力。
6. 现场反馈状态流转闭环。
7. 统计分析大屏。

## 4. V1.1 规划

1. 现场反馈闭环。
2. 统计指标口径和分析页。
3. C# 执行器接入。
4. 结构树复制、恢复和版本管理。
5. 报告归档权限细化。

## 5. V2.0 规划

1. 文件中心。
2. 异步队列。
3. 独立前端。
4. 更多炉型模板。

## 6. AI 开发边界

1. 代码生成必须先满足 V1.0。
2. V1.1 和 V2.0 只允许保留字段、接口占位或说明。
3. 涉及专业公式且缺少来源时必须标识为 Mock 或待专业校核。
4. 不得为了规划项扩大数据库和页面复杂度。
