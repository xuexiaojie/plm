# 07 页面需求规格

## 1. 页面清单

| 页面 | 路径 | 目的 |
|---|---|---|
| 门户页 | `/` | 导航和系统说明 |
| 项目页 | `/projects` | 项目列表和创建 |
| 名目页 | `/projects/{id}/items` | 名目列表和创建 |
| 结构树页 | `/items/{id}/tree` | 节点树维护 |
| 计算页 | `/nodes/{id}/execute` | 参数录入和执行 |
| 执行详情页 | `/executions/{id}` | 结果、日志、报告入口 |
| 审批页 | `/approvals` | 审批列表和操作 |
| 报告页 | `/reports/{id}` | 报告查看和下载 |
| 对比页 | `/comparisons` | 同类结果对比 |
| 资料管理页 | `/artifacts` | 现场反馈、审图单、技术附件、图纸目录录入和批量录入 |
| AI 分析页 | `/ai-analysis` | 同一设备计算结果与多类型资料联合分析 |

## 2. 通用 UI 要求

1. 顶部导航固定显示主要页面。
2. 表格必须有空状态、加载状态、错误状态。
3. 表单必填项必须标识。
4. 危险操作必须二次确认。
5. AI 分析在未配置大模型 API 时必须显示“演示分析”或 Mock Provider。

## 3. 项目页字段

表单字段：`code`、`name`、`owner_user_id`、`status`、`description`。

表格列：编码、名称、负责人、状态、创建时间、操作。

## 4. 名目页字段

表单字段：`code`、`name`、`furnace_type`、`business_scope`、`design_stage`、`description`。

表格列：编码、名称、炉型、业务范围、状态、最近执行、操作。

## 5. 结构树页

按钮：新增根节点、新增子节点、绑定模板、排序保存。

字段：节点名称、节点类型、模板、排序。

## 6. 计算页

1. 根据模板 `input_fields` 动态展示表单。
2. 执行后展示状态、输出字段、警告、错误、建议。
3. 不可行结果必须醒目标识。

## 7. 报告和审批页

1. 报告状态显示草稿、待审、正式、归档。
2. 审批动作按权限显示。
3. 正式报告才显示正式编号。

## 8. 资料管理页

1. 支持单条录入项目资料。
2. 支持按项目批量录入资料。
3. 资料类型固定为 `site_feedback`、`drawing_review`、`technical_attachment`、`drawing_catalog`。
4. 批量录入使用 JSON 输入，默认示例包含四类资料。
5. `project_item_id` 可为空，表示项目级资料。

## 9. AI 联合分析页

1. 支持输入设备名称，例如“装出钢机”。
2. 支持选择或输入执行 ID 列表。
3. 支持选择或输入资料 ID 列表。
4. 分析问题必须可编辑。
5. 页面展示 Provider、状态、摘要、风险点、矛盾点和建议。
