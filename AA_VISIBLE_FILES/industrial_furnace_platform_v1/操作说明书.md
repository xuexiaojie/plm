# 工业炉计算平台 V1.0 操作说明书

这份说明书按普通用户的使用顺序来写。你可以把它当作一条工作路线：先进入系统，再建项目，再做计算，再上传资料，最后用 AI 问答查资料、看结论、下报告。

当前预览地址：

```text
https://8000-1139be977bf349a2.monkeycode-ai.online
```

## 1. 先认识系统长什么样

打开系统后，先看到登录页。登录成功后，页面分成左右两块：左边是菜单，右边是当前功能页面。

```text
┌────────────────────────────────────────────────────────────┐
│ 工业炉计算平台 V1.0                                        │
├──────────────┬─────────────────────────────────────────────┤
│ 左侧菜单      │ 右侧功能页面                                  │
│              │                                             │
│ 项目管理      │ 这里显示项目台账、计算页面、资料页面、AI回答等   │
│ 计算管理      │                                             │
│ 审批报告      │                                             │
│ 数字孪生      │                                             │
│ 项目资料      │                                             │
│ AI 问答       │                                             │
│ 权限分配      │                                             │
└──────────────┴─────────────────────────────────────────────┘
```

左侧菜单可以理解成一本工具箱：

| 菜单 | 你用它做什么 |
| --- | --- |
| 项目管理 | 建项目、查项目、维护项目经理和企业信息 |
| 计算管理 | 选择项目和炉型，打开计算模块，得到计算结果和报告 |
| 审批报告 | 查看计算报告和审批状态 |
| 数字孪生 | 看炉型相关的结构和运行示意 |
| 项目资料 | 上传现场反馈、审图单、技术说明、图纸目录、材料表等资料 |
| AI 问答 | 像聊天一样问项目资料里的问题 |
| 权限分配 | 查看角色和权限配置 |

## 2. 一条最常用的工作路线

日常使用时，建议按这条路线走：

```text
登录系统
  ↓
项目管理：确认工程项目存在
  ↓
计算管理：选择项目和炉型，执行计算
  ↓
项目资料：上传现场反馈、审图单、技术说明等资料
  ↓
AI 问答：提问，快速查找项目资料里的问题和结论
  ↓
审批报告：查看报告和审批状态
```

这条路线的核心是“先选项目”。计算、资料和 AI 问答都围绕同一个工程项目工作。选错项目，后面的资料和回答就会跟着偏。

## 3. 登录系统

1. 打开预览地址或部署地址。
2. 在登录页输入用户名和密码。
3. 点击登录按钮。
4. 进入系统主界面后，左侧出现功能菜单。

```text
│ 用户名  [ demo           ]  │

```

## 4. 项目管理：先把工程项目整理好

进入 `项目管理` 后，你看到的是工程项目台账。这里记录项目名称、项目编码、企业、项目经理和项目介绍。

项目管理建议这样用：

1. 先查项目是否已经存在。
2. 项目存在时，确认项目经理、企业、项目介绍是否正确。
3. 项目不存在时，新建项目。
4. 项目介绍尽量写清楚炉型、产线、产能、改造范围、特殊边界条件。

项目卡片示意：

```text
┌────────────────────────────────────────────┐
│ 大马东钢加热炉项目                          │
│ 项目编码：PRJ-2026-001                      │
│ 企业：东钢集团                              │
│ 项目经理：张工                              │
│ 项目介绍：步进炉改造，重点复核炉门密封和风机能力 │
└────────────────────────────────────────────┘
```

项目介绍写得越清楚，后续模糊查询和 AI 问答越容易找到正确上下文。

## 5. 计算管理：按项目和炉型做计算

进入 `计算管理` 后，先选择工程项目，再选择炉型。系统固定提供 3 个炉型标签：`步进炉`、`辊底炉`、`环形炉`。

页面逻辑如下：

```text
┌──────────────────────────────────────────────────────────┐
│ 工程项目模糊查询：[ 输入项目名、项目经理、企业、介绍 ]       │
│ 工程项目：[ 请选择工程项目                 v ]              │
│ 项目信息：项目经理、企业、项目介绍                          │
├──────────────────────────────────────────────────────────┤
│ 炉型标签：[ 步进炉 ] [ 辊底炉 ] [ 环形炉 ]                  │
├──────────────────────────────────────────────────────────┤
│ 计算模块按钮区                                             │
│ [ 加热曲线 ] [ 传热计算 ] [ 排烟计算 ] [ 水梁计算 ] ...     │
└──────────────────────────────────────────────────────────┘
```

操作步骤：

1. 在 `工程项目模糊查询` 中输入关键词。
2. 在 `工程项目` 下拉框里选择目标项目。
3. 查看项目经理和项目介绍，确认项目选对。
4. 点击炉型标签，例如 `步进炉`。
5. 点击具体计算模块，例如 `加热曲线`。
6. 在弹窗里填写参数。
7. 点击计算。
8. 查看结果、图示和建议。
9. 下载 PDF 计算报告。

未选择工程项目前，炉型和计算按钮会变灰。这个设计是为了避免计算结果落到错误项目下。

## 6. 计算弹窗：填参数、看结果、下报告

大多数计算模块都会弹出一个计算窗口。窗口里通常有三块：参数输入、计算按钮、结果输出。

```text
┌──────────────────────────────────────────────┐
│ 加热曲线计算                                  │
│ 工程项目：大马东钢加热炉项目                   │
├──────────────────────────────────────────────┤
│ 参数输入                                      │
│ 厚度：[ 120 ] mm                              │
│ 装炉温度：[ 25 ] ℃                            │
│ 目标出炉温度：[ 1180 ] ℃                      │
│ 在炉时间：[ 180 ] min                         │
├──────────────────────────────────────────────┤
│ [ 计算 ] [ 下载 PDF 计算报告 ]                 │
├──────────────────────────────────────────────┤
│ 结果输出                                      │
│ 最终平均温度：1178 ℃                          │
│ 表心温差：18 ℃                                │
│ 判定：可行                                    │
│ 图示：升温曲线、温度云图、设备示意              │
└──────────────────────────────────────────────┘
```

结果页会给出：

| 内容 | 说明 |
| --- | --- |
| 判定状态 | 告诉你当前参数是否可行 |
| 关键指标 | 温度、流量、强度、压损等核心结果 |
| 图示输出 | 曲线图、结构图、云图或系统示意图 |
| 工程建议 | 提醒需要复核或调整的地方 |
| PDF 报告 | 用于归档、汇报、审批 |

## 7. 项目资料：上传资料给 AI 看

进入 `项目资料` 后，先选择工程项目，再上传资料。资料会按项目归档，AI 问答也只查当前项目下的资料。

页面示意：

```text
┌──────────────────────────────────────────────────────────┐
│ 项目资料模糊查询：[ 输入项目名、项目经理、企业、介绍 ]       │
│ 工程项目：[ 大马东钢加热炉项目              v ]              │
│ 项目信息：项目经理、项目介绍                                │
├──────────────────────────────────────────────────────────┤
│ 资料类型：[ 现场反馈 v ]                                   │
│ 条目标题：[ 大马东钢现场反馈 ]                              │
│ 上传日期：[ 2026-06-10 ]                                   │
│ 上传人：[ 当前用户 ]                                       │
│ 资料内容：[ 这里填写简要说明 ]                              │
│ 上传文档、图片、视频：[ 选择文件 ]                          │
│ [ 确认上传 ]                                               │
└──────────────────────────────────────────────────────────┘
```

支持的资料类型：

| 资料类型 | 常见内容 |
| --- | --- |
| 现场反馈 | 现场问题、运行异常、整改意见 |
| 审图单 | 审图意见、设计校审问题 |
| 技术说明 | 设计说明、计算边界、方案说明 |
| 图纸目录 | 图纸编号、版本、专业分类 |
| 材料表 | 材料清单、设备清单、规格型号 |
| 专利等技术文档 | 专利、论文、外部技术资料 |

上传后系统会做两件事：

1. 保存资料记录。
2. 对 `.docx` 和文本类文件读取正文，切成小片段，生成本地检索数据。

资料上传提示：

| 文件类型 | 当前处理方式 |
| --- | --- |
| `.docx` | 自动解析 Word 正文 |
| `.txt`、`.md`、`.csv`、`.json` 等文本文件 | 自动读取正文 |
| PDF、Excel、图片、视频 | 记录附件信息，当前不解析正文 |

如果你想让 AI 回答某个 Word 文档里的内容，请优先上传 `.docx` 文件。

## 8. AI 问答：像聊天一样查项目资料

进入 `AI 问答` 后，在提问框里直接写问题，例如：

```text
大马东钢项目出现过什么问题？
```

点击 `提问` 后，系统会按当前工程项目执行这条链路：

```text
读取当前项目
  ↓
检索项目资料切片
  ↓
找到相关现场反馈、审图单、技术说明等片段
  ↓
调用 AI 生成回答
  ↓
在页面显示回答
```

AI 问答页面示意：

```text
┌──────────────────────────────────────────────────┐
│ AI 问答                                           │
│ 像 ChatGPT 一样提问，系统只查当前项目资料。         │
├──────────────────────────────────────────────────┤
│ 提问                                              │
│ [ 大马东钢项目出现过什么问题？                  ] │
│ [ 提问 ]                                          │
├──────────────────────────────────────────────────┤
│ 回答                                              │
│ 根据项目资料，现场反馈中提到炉门密封不严，         │
│ 需要复核风机能力，并检查相关设计边界。             │
└──────────────────────────────────────────────────┘
```

提问建议：

| 问法 | 效果 |
| --- | --- |
| `大马东钢项目出现过什么问题？` | 查项目问题总览 |
| `炉门密封有什么反馈？` | 查具体问题 |
| `审图单里有哪些意见？` | 查审图资料 |
| `技术说明里风机能力怎么写的？` | 查技术边界 |
| `材料表里有哪些关键设备？` | 查清单内容 |

如果回答说资料不足，先回到 `项目资料` 上传 `.docx` 或文本资料，再重新提问。

## 9. 审批报告：看结果是否闭环

`审批报告` 用来查看计算结果、报告状态和审批流信息。常见使用方式：

1. 在计算弹窗里完成计算。
2. 生成或下载 PDF 计算报告。
3. 到 `审批报告` 查看报告状态。
4. 审批人员确认结果，必要时退回修改。

审批报告关注三件事：

| 关注点 | 说明 |
| --- | --- |
| 计算结果 | 参数和输出是否合理 |
| 报告状态 | 报告是否已经生成 |
| 审批状态 | 是否通过、退回或待处理 |

## 10. 数字孪生：看结构和运行示意

`数字孪生` 主要用于辅助理解炉型结构和运行状态。它适合在计算或汇报时打开，用来配合说明炉膛、钢坯、辊道、水梁、烧嘴等对象之间的关系。

示意逻辑：

```text
┌────────────────────────────────────────────┐
│ 炉体结构                                    │
│ 入口 → 预热段 → 加热段 → 均热段 → 出口       │
│ 钢坯位置、温度变化、设备区域                 │
└────────────────────────────────────────────┘
```

## 11. 权限分配：确认谁能做什么

`权限分配` 用来查看角色和权限。普通用户主要关注自己是否有以下权限：

| 权限 | 作用 |
| --- | --- |
| 项目管理 | 新建和维护工程项目 |
| 计算执行 | 执行计算模块 |
| 报告下载 | 下载计算报告 |
| 资料管理 | 上传和维护项目资料 |
| AI 问答 | 使用项目资料问答 |

如果某个按钮点不了，先检查是否选了工程项目，再检查当前角色是否有对应权限。

## 12. 常见问题和处理办法

| 现象 | 处理办法 |
| --- | --- |
| 炉型按钮是灰色 | 先在 `计算管理` 选择工程项目 |
| 资料上传区域是灰色 | 先在 `项目资料` 选择工程项目 |
| AI 回答找不到内容 | 先上传 `.docx` 或文本资料，再重新提问 |
| AI 查询很慢 | 确认资料列表只显示摘要；提问时等待回答区刷新 |
| Word 内容没有被回答 | 上传 `.docx` 格式，避免上传旧版 `.doc` |
| PDF 里的内容查不到 | 当前 PDF 只记录附件信息，请把关键文字补充到资料内容或上传 `.docx` |
| 计算结果不合理 | 回到计算弹窗检查输入参数、单位和炉型 |

## 13. 推荐使用习惯

1. 每次操作前先确认工程项目。
2. 项目介绍尽量写清楚关键背景。
3. 现场反馈和审图意见用简短标题归档。
4. Word 资料优先保存为 `.docx` 再上传。
5. AI 提问尽量带项目名、设备名或问题关键词。
6. 计算完成后及时下载 PDF 报告。
7. 审批前先核对输入参数和图示结果。

## 14. 一次完整操作示例

假设你要处理 `大马东钢加热炉项目` 的现场问题，可以这样做：

1. 登录系统。
2. 点击 `项目管理`，确认 `大马东钢加热炉项目` 已存在。
3. 点击 `计算管理`，选择 `大马东钢加热炉项目`。
4. 选择 `步进炉`。
5. 打开 `加热曲线` 或相关计算模块。
6. 填写参数并点击计算。
7. 查看图示和计算建议。
8. 点击 `项目资料`。
9. 选择同一个工程项目。
10. 上传 `大马东钢现场反馈.docx`。
11. 点击 `AI 问答`。
12. 输入 `大马东钢项目出现过什么问题？`。
13. 查看回答，并根据回答回到计算或资料页面继续复核。

这套流程的目标是让项目、计算、资料和 AI 回答围绕同一个工程项目闭环，减少人工翻文档和反复查表的时间。
