# 计算与现场工作台操作说明书

## 1. 文档目的

本文档用于指导业务人员、实施人员和开发联调人员使用当前版本的计算与现场工作台，完成从基础录入、计算执行、审批查询到 AI 分析的完整操作。

## 2. 系统概览

当前系统由 FastAPI 提供接口和轻量页面，业务入口分为 4 个页面：

1. `/` 门户页：查看系统定位和进入各工作台。
2. `/entry` 录入台：录入项目、全局参数、现场反馈。
3. `/compute` 计算台：按项目和名目执行计算树，查看执行结果。
4. `/analysis` 分析台：查看项目结构、现场反馈、审批记录和 AI 分析结果。

系统围绕“项目 -> 名目 -> 树节点”组织计算，同时把现场反馈并入同一项目工作台。

## 3. 启动方式

### 3.1 安装依赖

```bash
pip install --break-system-packages -r requirements.txt
```

### 3.2 执行数据库迁移

```bash
alembic upgrade head
```

### 3.3 启动服务

```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

启动后可访问：

- `http://127.0.0.1:8000/`
- `http://127.0.0.1:8000/entry`
- `http://127.0.0.1:8000/compute`
- `http://127.0.0.1:8000/analysis`

## 4. 页面职责

### 4.1 门户页

门户页用于进入各工作区，适合第一次进入系统时确认当前版本的页面结构。

### 4.2 录入台

录入台当前支持以下数据录入：

1. 项目
2. 全局参数
3. 现场反馈

现场反馈支持绑定到：

1. 项目级
2. 名目级
3. 节点级

### 4.3 计算台

计算台用于：

1. 选择项目
2. 选择项目下名目
3. 查看当前名目的树节点
4. 执行整棵树
5. 查看最近执行结果

### 4.4 分析台

分析台用于汇总查看：

1. 项目结构树
2. 项目参数
3. 现场反馈
4. 审批记录
5. AI 分析结果

分析台优先通过项目工作台聚合接口加载统一上下文。

## 5. 推荐操作流程

推荐按以下顺序使用系统：

1. 在录入台创建项目。
2. 录入全局参数。
3. 录入项目相关现场反馈。
4. 通过 API 创建名目、树节点、计算步骤和输入引用。
5. 在计算台执行整树计算。
6. 在分析台查看现场反馈、审批和 AI 分析结果。

## 6. 录入台操作说明

### 6.1 创建项目

在 `/entry` 页面填写项目信息后提交。

当前项目至少包含以下信息：

1. `name`
2. `owner_user_id`
3. `status`

创建成功后，项目会显示在页面项目列表中。

### 6.2 创建全局参数

在录入台填写全局参数信息后提交。

适用场景：

1. 多项目共享的阈值
2. 标准工艺参数
3. 统一配置项

### 6.3 创建现场反馈

在录入台填写现场反馈表单。

建议填写字段：

1. `project_id`
2. `source`
3. `severity`
4. `title`
5. `content`
6. `reported_by`
7. `project_item_id`，适用于名目级反馈
8. `node_id`，适用于节点级反馈

### 6.4 反馈绑定规则

系统会校验反馈绑定的一致性：

1. 节点必须属于当前项目。
2. 当同时传 `project_item_id` 和 `node_id` 时，该节点必须属于该名目。

适合这样理解：

1. 只填 `project_id`：项目级反馈
2. 填 `project_id + project_item_id`：名目级反馈
3. 填 `project_id + project_item_id + node_id`：节点级反馈

## 7. 计算台操作说明

### 7.1 计算前准备

当前页面侧重执行，建模数据需要提前具备：

1. 项目已经创建
2. 项目下至少存在一个名目
3. 名目下已经建立树节点
4. 计算节点已绑定 `calc_step_id`

### 7.2 执行整树计算

操作步骤：

1. 打开 `/compute`
2. 选择项目
3. 选择名目
4. 点击执行整树

对应后端接口：

```text
POST /projects/{project_id}/items/{item_id}/run-tree
```

### 7.3 查看执行结果

执行完成后，可查看最近执行结果。

常用接口：

```text
GET /executions/{execution_id}
GET /executions/{execution_id}/results
```

结果中可关注：

1. 执行状态
2. 输出结果 `output_json`
3. 错误码 `error_code`
4. 输入快照 `input_snapshot_json`

### 7.4 输入上下文说明

节点执行时，系统会把现场反馈一并带入上下文，主要包含：

1. `feedback_context.project_feedback`
2. `feedback_context.item_feedback`
3. `feedback_context.node_feedback`

这意味着计算步骤可以读取当前项目、当前名目和当前节点的现场反馈背景。

## 8. 分析台操作说明

### 8.1 查看项目工作台聚合信息

打开 `/analysis` 后选择项目，系统会优先读取聚合接口：

```text
GET /projects/{project_id}/workspace
```

该接口会聚合：

1. 项目基本信息
2. 项目结构
3. 项目参数
4. 现场反馈
5. 审批信息
6. AI 分析信息

### 8.2 查看现场反馈

分析台会展示当前项目的现场反馈列表，便于快速核对：

1. 问题来源
2. 严重程度
3. 所属名目或节点
4. 反馈内容

对应接口：

```text
GET /projects/{project_id}/feedback
```

### 8.3 查看审批

分析台可查看当前项目审批记录。

对应接口：

```text
GET /projects/{project_id}/approvals
GET /approvals/{approval_id}
GET /approvals/{approval_id}/logs
```

审批状态包括：

1. `pending`
2. `in_review`
3. `approved`
4. `rejected`

### 8.4 查看 AI 分析

当前 AI 分析结果支持三流理论结构化输出。

常用接口：

```text
POST /ai/analysis
GET /ai/analysis
GET /ai/analysis/{request_id}
GET /ai/analysis/{request_id}/result
```

返回结果中的 `raw_response_json` 重点关注：

1. `analysis_framework`
2. `three_flows`
3. `three_flow_state`
4. `feedback_summary`
5. `shared_feedback_scope_id`

## 9. 审批操作说明

### 9.1 创建审批请求

使用接口：

```text
POST /approvals
```

可传 `total_stages` 用于定义多级审批。

### 9.2 审批通过

使用接口：

```text
POST /approvals/{approval_id}/approve
```

审批通过规则：

1. 当前级通过且未到最后一级，状态进入 `in_review`
2. 最后一级通过，状态进入 `approved`

### 9.3 审批拒绝

使用接口：

```text
POST /approvals/{approval_id}/reject
```

任一级拒绝后状态进入 `rejected`。

## 11. 常见场景示例

### 11.1 新建一个可执行项目

步骤如下：

1. 在录入台创建项目。
2. 用接口创建该项目下名目。
3. 用接口创建计算步骤。
4. 用接口创建树节点，并把节点绑定到步骤。
5. 按需创建参数和输入引用。
6. 在计算台执行整树。
7. 在分析台查看结果和 AI 分析。

### 11.2 录入一条节点级现场反馈

步骤如下：

1. 确认项目、名目、节点都已存在。
2. 在录入台填写反馈。
3. 录入 `project_id`、`project_item_id`、`node_id`。
4. 提交后在分析台查看反馈是否出现在对应项目下。

### 11.3 发起一次 AI 分析

步骤如下：

1. 确认项目已有结构数据或执行结果。
2. 通过接口提交 AI 分析请求。
3. 在分析台或结果接口查看三流理论输出。

## 12. 常见问题排查

### 12.1 页面打开正常但没有数据

优先检查：

1. 数据库是否已迁移到最新版本
2. 当前数据库是否为正确的 `DATABASE_URL`
3. 是否已经创建项目和相关基础数据

### 12.2 执行失败

优先检查：

1. 节点是否绑定了 `calc_step_id`
2. 步骤的 `language` 是否为当前支持值
3. Python 脚本是否返回合法 JSON
4. 外部 C# 产物路径是否存在

当前错误码包括：

1. `timeout`
2. `runtime_error`
3. `invalid_output`
4. `artifact_missing`
5. `unsupported_language`

### 12.3 审批接口报数据库字段问题

执行一次迁移：

```bash
alembic upgrade head
```

### 12.4 反馈绑定失败

优先检查：

1. `node_id` 是否属于当前项目
2. `node_id` 是否属于当前 `project_item_id`

## 13. 当前版本边界

当前版本已具备项目闭环能力，仍有以下边界：

1. 录入台还没有项目参数、名目、树节点的完整表单化录入
2. 计算台还未提供单节点执行入口
3. 分析台还未提供完整对比报表面板
4. AI provider 当前仍以 stub 为主

## 14. 维护建议

1. 每次升级代码后执行 `alembic upgrade head`
2. 新增字段时优先补正式迁移脚本
3. 调整页面能力时同步更新本说明书
4. 对外演示时优先按“录入 -> 计算 -> 分析”顺序操作
